## Data Storage Model

ML Lab stores data in the user's browser using localStorage. No backend database is required.

## Data Types

| Data | Storage | Purpose |
|---|---|---|
| Lesson progress | localStorage | Continue learning |
| Notes and scratchpad | localStorage | Personal study notes |
| Bookmarks | localStorage | Revision shortcuts |
| Quiz scores | localStorage | Local leaderboard |
| Timer history | localStorage | Study discipline |
| Cohort records | localStorage | Local classroom tracking |
| Audit log | localStorage | Local operational transparency |
| Backup files | User-downloaded JSON | Migration and recovery |

## Data Protection Guidelines

1. Avoid storing sensitive personal data.
2. Explain localStorage usage before classroom deployment.
3. Export backups before clearing browser data.
4. Use HTTPS hosting for public deployment.
5. Do not connect paid AI APIs unless a future project explicitly budgets for it.

## No AI API Declaration

ML Lab does not send data to external AI services. All mentoring, explanations, calculators and templates are local and rule-based.
