## Full Feature Explanation

### 1. Dashboard
The dashboard is the learner's command centre. It shows the platform purpose, Adewale/HMG persona banner, progress summary, study streak and all modules. It also provides quick links to Portfolio, HMG Concepts, GitHub and WhatsApp.

### 2. Learn Module
The Learn Module contains 32 progressive annotated scikit-learn, statsmodels, NLP, deployment and responsible ML lessons. Each lesson includes commented Python code, expected output, a detailed explanation panel, parameter breakdown, common mistakes and a knowledge-check quiz. This is designed to teach *why the code works*, not just what to type.

### 3. Practice Lab
The Practice Lab uses 16 fill-in-the-blank coding exercises. Learners complete real ML code patterns such as `train_test_split`, `StandardScaler`, `RandomForestClassifier`, `Pipeline`, `GridSearchCV`, statsmodels OLS and joblib save/load. Instant feedback turns passive reading into active recall.

### 4. Model Playground
The Model Playground lets users adjust hyperparameters using sliders and dropdowns. The Python code updates live, helping learners understand how parameters such as `n_estimators`, `max_depth`, `C`, `learning_rate`, `kernel`, `alpha` and `class_weight` affect model setup.

### 5. Cheat Sheet
The Cheat Sheet is a searchable quick reference for key scikit-learn and statsmodels APIs. It covers data splitting, preprocessing, pipelines, classification models, regression models, metrics, hyperparameter search, statsmodels and model persistence.

### 6. Glossary
The Glossary explains ML terminology in plain English with code examples. It is useful when a learner sees terms such as data leakage, F1 score, AUC, p-value, Pipeline, fit/transform, regularisation or feature importance.

### 7. Error Decoder
The Error Decoder is a local troubleshooting tool. The user pastes common scikit-learn or Python errors, and the platform explains the meaning, root cause and solution. Covered examples include NotFittedError, shape mismatch, ConvergenceWarning, unknown categories, roc_auc_score misuse, GridSearchCV pipeline parameters and statsmodels `add_constant` mistakes.

### 8. Scratchpad
The Scratchpad is a persistent multi-tab notepad. Users can write notes, draft Python code, insert common snippets, copy all content or export a tab as a `.py` file. It saves automatically to browser localStorage.

### 9. Model Comparison
The Model Comparison page compares popular models by task, scaling requirement, speed, accuracy potential, interpretability, overfitting risk and imbalance handling. It helps learners choose models deliberately instead of guessing.

### 10. Learning Path
The Learning Path gives a structured roadmap from foundation to deployment. It uses prerequisites so learners progress from train/test split and estimator API through preprocessing, pipelines, modelling, evaluation, selection and model saving.

### 11. Quiz Arena
The Quiz Arena tests knowledge under time pressure. It includes multiple timed modes, countdowns, grading, answer review and a local leaderboard. Scores are stored only in the browser.

### 12. Formula Reference
The Formula Reference connects mathematics to code. It explains classification metrics, regression metrics, regularisation, bias-variance, z-score scaling, sigmoid, Gini impurity and cross-entropy with formulas, variable definitions, plain-English interpretation and sklearn code.

### 13. Bookmarks
Users can save important lessons, formulas, glossary terms and cheat-sheet entries. The Bookmark Manager allows filtering, searching, exporting and jumping back to saved items.

### 14. Session Timer
The Session Timer supports stopwatch and Pomodoro study sessions. Users can label study time, save history and review today's total, all-time total, longest session and average session. It supports disciplined learning habits.

### 15. Progress Tracker
The Progress Tracker shows lesson completion, exercise completion, section progress bars, milestone badges and exportable progress reports. It also allows manual toggling and reset with confirmation.

### 16. Feature Guide
The Feature Guide is a new in-system documentation page. It explains every platform module using four questions: what it is, why it exists, how it works and when to use it.

### 17. Project Builder
The Project Builder generates a practical local ML project blueprint. Users choose task type, domain, imbalance expectation and deployment target. The system returns a recommended metric strategy, model family, notebook structure, expert checks and starter code.

### 18. Local Mentor
The Local Mentor gives AI-like help without an AI API. It uses transparent keyword rules for common questions about leakage, ROC-AUC, imbalance, pipelines, categorical encoding, deployment, statsmodels and model choice. Because it is local and rule-based, it is free and privacy-preserving.

### 19. Deployment Guide
The Deployment Guide gives clear instructions for publishing the platform using free tools. It covers GitHub Pages, Cloudflare Pages, Netlify Drop and local/offline usage.

### 20. Visioner Profile
The Visioner Profile embeds Adewale Samson Adeagbo's professional identity, HMG Concepts background, contact details, links, current focus and platform philosophy directly inside the app.

### 21. Metric Calculator
The Metric Calculator helps learners understand model evaluation numerically. For classification, users enter TP, TN, FP and FN, and the system calculates Accuracy, Precision, Recall, Specificity, F1, False Positive Rate and False Negative Rate. For regression, users enter actual and predicted values, and the system calculates MAE, MSE, RMSE and R².

### 22. CSV Inspector
The CSV Inspector allows users to paste or upload a CSV file and profile it locally. It reports row count, column count, missing values, inferred column types, distinct counts, numeric minimum, mean and maximum values, plus a preview table. No data is uploaded to any server.

### 23. Flashcards
The Flashcards module creates review cards from glossary terms, formulas and lessons. Users can reveal answers, mark cards as hard or mastered, and track review status locally. This supports spaced repetition and active recall.

### 24. Backup Center
The Backup Center exports all browser-stored platform data as a JSON file and imports it later. This protects notes, progress, bookmarks, leaderboard, timer history and flashcard status when users change devices or clear browser cache.

### 25. ML Workflow Checklist
The ML Workflow Checklist turns professional modelling practice into a repeatable process. It covers problem framing, data audit, EDA, splitting, preprocessing, modelling, evaluation, interpretation, deployment and monitoring. It is useful before publishing a portfolio project or presenting a model.

### 26. Interview Prep
The Interview Prep module contains practical ML questions and clear answers. It helps learners explain metrics, leakage, pipelines, preprocessing, model choice, tuning, statsmodels and deployment in simple language.

### 27. Enterprise Center
The Enterprise Center gives the platform an admin-style operational layer. It shows readiness score, local storage usage, audit event count, organisation profile, free-tool architecture and risk register. It helps schools, partners and teams understand how the platform can be operated responsibly without paid enterprise software.

### 28. Cohort Manager
The Cohort Manager provides local LMS-lite functionality. Admins can add learner records, assign cohorts and tracks, record status, track progress and export reports as CSV for Google Sheets, Excel or Power BI. It uses localStorage only and does not require a database.

### 29. Governance Center
The Governance Center documents privacy, security, accessibility and deployment responsibilities. It includes checklists, a data map, retention notes and a clear no-AI-API declaration for institutional confidence.

### 30. Audit Log
The Audit Log stores selected local events such as learner additions, governance toggles, exports and progress completions. It is not a server-grade tamper-proof audit system, but it gives transparent local operational history for demos and classroom administration.

### 31. PWA Offline Support
The project now includes `manifest.webmanifest`, `sw.js` and `icon.svg`. When deployed over HTTPS, supported browsers can cache the app for offline use and allow installation-like behaviour. This is implemented with free static files only.

### 32. Curriculum Hub
The Curriculum Hub is a full beginner-to-expert data science syllabus. It structures the platform into phases, weekly pacing, outcomes, lessons, assessments, projects and capstone options. It now supports 32 progressive lessons and 16 practice exercises.

### 33. Setup Wizard
The Setup Wizard prepares ML Lab for schools, cohorts, workshops and partner deployments. It collects institution, audience, hosting, admin and data-policy details, then generates a downloadable deployment brief.

### 34. Issue Desk
The Issue Desk is a local helpdesk alternative. It tracks bugs, support requests, curriculum suggestions, deployment tasks and feature requests without paid support software. Tickets can be exported as CSV.

### 35. Resource Library
The Resource Library provides copy-ready templates for project README files, model cards, requirements.txt, Streamlit apps, GitHub issues and lesson plans. This helps learners and admins produce professional documentation quickly.

### 36. Release Manager
The Release Manager provides a version-readiness checklist and release notes. It helps admins confirm that documentation, governance, PWA files and deployment steps are ready before publishing a new version.

## Local Usage

### Option A — Direct Browser Open

1. Download or copy the project folder.
2. Open the folder.
3. Double-click `index.html`.
4. The platform opens in your browser.

### Option B — Local Server

Use this if your browser restricts localStorage for files opened through `file://`.

```bash
cd ml-learning-platform
python -m http.server 8080
```

Then open:

```text
http://localhost:8080
```

## Free Deployment Process — GitHub Pages

### Step 1 — Prepare Files
Ensure these files are in the project root:

```text
index.html
README.md
package.json
```

`index.html` must be at the root, not inside a subfolder.

### Step 2 — Create Repository

1. Go to <https://github.com/new>
2. Repository name: `ml-learning-platform`
3. Visibility: Public
4. Do not add a README if you already have this one locally.
5. Click **Create repository**.

### Step 3 — Push to GitHub

```bash
cd ml-learning-platform
git init
git add .
git commit -m "Enhance ML learning platform"
git branch -M main
git remote add origin https://github.com/cssadewale/ml-learning-platform.git
git push -u origin main
```

Replace `cssadewale` and repository name if needed.

### Step 4 — Enable GitHub Pages

1. Open the repository on GitHub.
2. Go to **Settings**.
3. Click **Pages**.
4. Source: **Deploy from a branch**.
5. Branch: **main**.
6. Folder: **/ (root)**.
7. Click **Save**.

### Step 5 — Open the Live Site
Wait 1–2 minutes. Your URL should look like:

```text
https://cssadewale.github.io/ml-learning-platform/
```

### Step 6 — Update Later

```bash
git add .
git commit -m "Update lessons and tools"
git push
```

GitHub Pages redeploys automatically.

## Free Deployment Process — Cloudflare Pages

1. Push the files to GitHub.
2. Go to <https://dash.cloudflare.com>
3. Open **Workers & Pages**.
4. Click **Create application**.
5. Choose **Pages** → **Connect to Git**.
6. Select the repository.
7. Build settings:
   - Framework preset: **None**
   - Build command: leave blank
   - Build output directory: `/` or blank/root
8. Click **Save and Deploy**.
9. Cloudflare gives a free `.pages.dev` URL.

---

## Free Deployment Process — Netlify Drop

1. Ensure `index.html` is at the top level of the folder.
2. Go to <https://app.netlify.com/drop>
3. Drag the project folder onto the page.
4. Netlify uploads and hosts it.
5. Rename the site under Site Settings if desired.

This method is fast for demos, but GitHub-connected deployment is better for long-term updates.

## Troubleshooting

| Problem | Cause | Fix |
|---|---|---|
| 404 after deploy | Pages not ready or wrong branch | Wait 2 minutes and confirm main/root settings |
| White screen | `index.html` missing or JavaScript syntax issue | Keep `index.html` at root and check browser console |
| Changes not showing | Browser cache | Hard refresh with Ctrl+Shift+R |
| Fonts not loading offline | Google Fonts needs internet | App still works with fallback fonts |
| Progress not saving locally | Browser blocks localStorage under `file://` | Use `python -m http.server 8080` |
| Git push fails | Wrong remote URL | Run `git remote -v` and correct the URL |
